NoSQL

¿Qué son las bases de datos no relacionales?

Las bases de datos no relacionales, también llamadas NoSQL, son sistemas de almacenamiento de información que no utilizan tablas tradicionales como las bases de datos relacionales. En lugar de trabajar con filas y columnas, almacenan la información en formatos más flexibles, como:

  • Documentos.
  • Clave-valor.
  • Grafos.
  • Columnas.

Características principales

  • Flexibilidad en la estructura de datos.
  • Alta escalabilidad.
  • Buen rendimiento con grandes volúmenes de información.
  • Adecuadas para aplicaciones web modernas y Big Data.

Diferencias entre bases de datos relacionales y no relacionales

Tipos de bases de datos NoSQL

Bases de datos orientadas a documentos

Guardan información en documentos JSON o similares. Ejemplo: MongoDB

Bases de datos clave-valor

Almacenan pares clave → valor. Ejemplo: Redis

Bases de datos orientadas a grafos

Representan relaciones entre datos. Ejemplo: Neo4j

Bases de datos por columnas

Organizan la información por columnas en lugar de filas. Ejemplo: Apache Cassandra

Ventajas

  • Mayor velocidad en ciertos procesos.
  • Escalabilidad horizontal.
  • Flexibilidad para manejar datos no estructurados.
  • Muy usadas en aplicaciones modernas.

Desventajas

  • Menor estandarización.
  • Algunas no soportan relaciones complejas.
  • Menor uso de transacciones avanzadas comparado con SQL.

MongoDB

¿Qué es MongoDB?

MongoDB es una base de datos NoSQL orientada a documentos. Fue desarrollada por MongoDB y almacena la información en documentos tipo JSON llamados BSON.

Características de MongoDB

Modelo orientado a documentos
Los datos se almacenan en documentos similares a JSON.

Ejemplo


{
  "nombre": "Juan",
  "edad": 25,
  "ciudad": "Bogotá"
}

Esquema flexible

No es obligatorio que todos los documentos tengan los mismos campos.

Escalabilidad

MongoDB puede manejar grandes cantidades de datos distribuidos en múltiples servidores.

Alto rendimiento

Permite consultas rápidas y eficientes.

Compatibilidad con múltiples lenguajes

Puede usarse con:

  • JavaScript
  • Python
  • Java
  • PHP
  • Node.js

Mongoose

¿Qué es Mongoose?

Mongoose es una librería para Node.js que facilita trabajar con MongoDB. Mongoose funciona como un ODM (Object Data Modeling), permitiendo modelar los datos mediante esquemas y clases.

¿Para qué sirve?

  • Conectar aplicaciones Node.js con MongoDB.
  • Crear esquemas.
  • Validar datos.
  • Facilitar consultas a la base de datos.

Funcionamiento básico

Instalar Mongoose


npm install mongoose

Conexión a MongoDB


const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/mibasedatos')
.then(() => console.log('Conectado'))
.catch(err => console.log(err));

Crear un esquema


const usuarioSchema = new mongoose.Schema({
  nombre: String,
  edad: Number
});

Crear un modelo


const Usuario = mongoose.model('Usuario', usuarioSchema);

Guardar datos


const nuevoUsuario = new Usuario({
  nombre: 'Juan',
  edad: 25
});

nuevoUsuario.save();

Ventajas de Mongoose

  • Validación automática de datos.
  • Organización del código.
  • Manejo sencillo de consultas.
  • Integración completa con Node.js.
